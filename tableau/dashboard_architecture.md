## Dashboard Architecture — PCOSense

PCOSense uses a three-dashboard architecture designed to move users
from awareness → understanding → action.

Each dashboard serves a specific decision-making role.

---

## Dashboard 1 — Executive Health Overview

### Audience
- Hackathon judges
- Healthcare program managers
- Product stakeholders

### Core Question
“Is there a risk problem, and how serious is it?”

### Key KPIs
- Average PCOS Risk Index
- % High Risk Users
- Risk Trend (Last 6 Months)
- Most Common Risk Driver

### Visual Components
- KPI tiles (top)
- Risk category donut
- Time-series risk trend
- Driver contribution bar chart

### Why This Dashboard Matters
- Instant clarity in under 10 seconds
- Ideal for demo opening
- Sets emotional + analytical context

---

## Dashboard 2 — Risk Explorer (Interactive Analysis)

### Audience
- Analysts
- Clinicians
- Power users

### Core Question
“Why is this risk happening?”

### Interactive Controls
- Risk Category Filter
- Age Group Filter
- BMI Range Selector
- Lifestyle Factor Toggle

### Visual Components
- Heatmap of risk drivers
- Scatter plot (Lifestyle vs Metabolic Risk)
- Symptom frequency bars
- Demographic segmentation view

### Why This Dashboard Matters
- Shows analytical depth
- Demonstrates Tableau interactivity
- Highlights semantic modeling strength

---

## Dashboard 3 — Action & Alerts Console

### Audience
- End users
- Wellness coaches
- Integrated systems (Slack / Salesforce)

### Core Question
“What should be done now?”

### Visual Components
- High-risk user table
- Triggered alert summary
- Recommended action cards
- Risk change indicators

### Alert Logic Example
- PCOS Risk Index > 70
- Rapid increase (>15 points in 30 days)
- Lifestyle risk persistence

### Why This Dashboard Matters
- Converts analytics into action
- Strong alignment with actionable analytics prize
- Perfect segue into integrations demo

---

## Design Principles Applied

- One primary question per dashboard
- Minimal cognitive load
- Color used only for meaning
- Progressive disclosure of complexity
